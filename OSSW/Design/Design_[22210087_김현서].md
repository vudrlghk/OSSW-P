# 1. Introduction

본 문서는 판타지 세계관을 배경으로 하는 턴제 로그라이크 게임 **'Echoes of Legacy'**의 소프트웨어 설계 명세서이다. 본 문서는 Analysis 단계에서 정의한 요구사항과 개념 모델을 바탕으로, 구현 단계에서 직접 참조 가능한 정적/동적 설계를 제시하는 것을 목적으로 한다.

'Echoes of Legacy'는 일반적인 단일 주인공 중심의 로그라이크 장르와 차별화되는 **다중 캐릭터 순차 플레이(Multi-character legacy system)**를 핵심으로 한다. 플레이어가 조작한 이전 캐릭터의 성장, 생존, 명성치, 선택 결과가 세계(World) 데이터에 누적되어 후속 캐릭터의 탐험, 이벤트 발생 조건, NPC의 대사, 그리고 파티 구성에까지 지속적인 영향을 미친다.

본 게임은 화려한 3D 그래픽이나 복잡한 액션보다는 텍스트와 정적 이미지 중심의 인터페이스를 채택하였다. 이는 서사적 몰입감을 극대화함과 동시에 학기 프로젝트라는 제한된 개발 환경에서 모듈성 및 구현 실현 가능성을 높이기 위함이다.

# 2. Class Diagram

## 2.1 설계 원칙 및 아키텍처 구조

본 시스템은 Unity 엔진의 특성을 고려하되, 객체 지향적 설계 원칙에 따라 명확한 **책임 분리(Separation of Concerns)**를 지향한다. 설계의 주요 원칙은 다음과 같다.

- **UI와 비즈니스 로직의 분리:** 화면 렌더링 및 사용자 입력(UI Layer)은 내부 데이터 연산(Control/Manager Layer)과 분리하여, UI 변경이 비즈니스 로직에 영향을 주지 않도록 한다.
- **레거시 데이터와 현재 진행 데이터의 분리:** 현재 조작 중인 개별 캐릭터의 데이터(`Character`, `Inventory`)와 누적되어 후속 플레이에 영향을 미치는 전역 데이터(`WorldState`, `ReputationSystem`)를 분리하여 설계한다.
- **계층화된 아키텍처:** 시스템을 UI Layer, Control/Manager Layer, Domain Model Layer, Persistence Layer의 4가지 계층으로 분류하여 모듈성을 확보한다.

전체 게임의 상태 및 주요 비즈니스 흐름을 제어하는 핵심 컨트롤러 클래스이다.

| Relationships | 모든 UI 클래스로부터 입력을 수신하며, Domain Model 객체(`WorldState`, `Character` 등)의 비즈니스 로직을 호출한다. |

|---|---|

# 3. Sequence Diagram

본 장에서는 객체 간의 동적 상호작용을 시간적 흐름에 따라 모델링한 시퀀스 다이어그램을 제시한다. 각 시나리오는 핵심 유스케이스를 기반으로 추출되었다.

## 3.1 Start New Game

![Start New Game](images/01_start_new_game.png)

**그림 3-1: Start New Game**

- **시나리오 설명:** 사용자가 메인 메뉴에서 새로운 게임을 시작할 때, 시스템이 기본 프로필과 월드 데이터를 초기화하고 캐릭터 선택 화면을 거쳐 탐험 화면으로 진입하는 과정이다.
- **참여 객체:** Player(액터), MainMenuUI, GameManager, PlayerProfile, WorldState, CharacterSelectUI
- **주요 메시지 흐름:**
  1. Player가 `MainMenuUI`에서 NewGame을 누르면 `GameManager.startNewGame()`이 호출된다.
  2. `GameManager`는 `PlayerProfile.createProfile()`과 `WorldState.initializeWorld()`를 차례로 호출하여 초기 환경을 구성한다.
  3. 초기화 완료 후 `GameManager`는 `CharacterSelectUI`를 통해 캐릭터 목록을 표시한다.
  4. Player가 특정 캐릭터를 선택(selectCharacter)하면 해당 캐릭터 정보가 설정되고, `openExploreScreen()`을 통해 최종 게임 화면으로 진입한다.

## 3.2 Explore Area

![Explore Area](images/02_explore_area)

**그림 3-2: Explore Area**

- **시나리오 설명:** 플레이어가 특정 지역을 탐험할 때, 월드의 Legacy 데이터(과거 기록)를 검사하여 특수 이벤트 발동 여부를 결정하고 화면에 갱신하는 과정이다.
- **참여 객체:** Player, ExploreUI, GameManager, Region, Event, WorldState, EventUI
- **주요 메시지 흐름 및 조건(Alt):**
  1. Player가 탐험 액션을 수행하면 `GameManager`가 `Region`으로부터 사용 가능한 `Event` 목록을 획득한다.
  2. `GameManager`는 `Event.checkCondition()`을 호출하고, 이는 다시 `WorldState.checkLegacyFlag()`를 통해 레거시 조건을 검증한다.
  3. **[alt: isEventTriggered == true]** 조건 만족 시, 이벤트 상세 데이터를 얻어 `EventUI`에 선택지와 상황을 표시(showEvent)한다.
  4. **[alt: isEventTriggered == false]** 조건 불만족 시 일반 탐험 텍스트만 `ExploreUI`에 갱신(updateExploreText)한다.

## 3.3 Save and Load Game

![Save and Load Game](images/06_save_load_game)

**그림 3-3: Save and Load Game**

- **시나리오 설명:** 플레이어가 현재까지의 진행 상황을 파일에 기록하거나, 기존 저장 파일로부터 이전 상태를 복원하는 과정이다. 레거시 데이터가 포함된 방대한 정보를 다루기 때문에 두 모델 객체로부터의 데이터 수집이 필수적이다.
- **참여 객체:** Player, SaveUI, GameManager, PlayerProfile, WorldState, SaveData
- **주요 메시지 흐름:**
  1. 저장(Save) 시, `GameManager`는 `PlayerProfile`(캐릭터 상태)과 `WorldState`(레거시 및 월드 상태)에서 각각 데이터를 수집한다.
  2. 수집된 데이터를 바탕으로 `SaveData.save()`를 호출하여 영구 저장소에 기록 후 UI에 결과를 통보한다.
  3. 불러오기(Load) 시, `GameManager`는 `SaveData.load(saveId)`를 호출하여 직렬화된 데이터를 받아 현재 상태로 복원(restoreCurrentState)하고 게임 화면을 연다.

## 3.4 Make Choice

![Make Choice](images/03_make_choice)

**그림 3-4: Make Choice**

- **시나리오 설명:** 이벤트 텍스트 내에서 플레이어가 특정 선택지를 골랐을 때, 해당 선택의 결과가 즉각적인 능력치 변화뿐만 아니라 레거시 데이터와 평판 시스템에 동시에 기록되는 과정이다.
- **참여 객체:** Player, EventUI, GameManager, Event, Choice, Character, WorldState, ReputationSystem
- **주요 메시지 흐름:**
  1. Player가 `EventUI`에서 옵션을 선택하면 `GameManager.applyChoice()`가 수행된다.
  2. `Choice.applyTo()`가 호출되어 현재 `Character`의 상태 업데이트와 `WorldState`의 Legacy Flag 갱신이 동시에 처리된다.
  3. 이후 `ReputationSystem.addReputation()`을 통해 누적 평판이 갱신되며, 모든 결과가 합산되어 `EventUI.showResult()`로 출력된다.

## 3.5 Battle Enemy

![Battle Enemy](images/04_battle_enemy)

**그림 3-5: Battle Enemy**

- **시나리오 설명:** 턴제 전투의 초기화, 턴 반복 루프, 적 사망 시 보상 획득 및 평판 갱신의 일련 과정을 나타낸다.
- **참여 객체:** Player, BattleUI, GameManager, BattleManager, Character, Enemy, ReputationSystem, WorldState
- **주요 메시지 흐름 및 조건(Loop/Alt):**
  1. `GameManager`가 `BattleManager.initializeBattle()`을 호출하여 전투를 시작한다.
  2. **[loop: BattleLoop]** Player가 액션을 선택하면 `Enemy`에게 데미지를 가하고 상태(enemyAlive)를 확인한다.
  3. **[alt: enemyAlive == true]** 적이 생존했으면 적의 AI 패턴을 선택하여 Player의 `Character`에 데미지를 가하고 턴을 넘긴다.
  4. **[alt: enemyAlive == false]** 적이 사망했으면 `ReputationSystem`에 보상을 추가하고 루프를 종료한다.
  5. 최종적으로 전투 결과를 `BattleUI`를 통해 표시한다.

## 3.6 Reflect Legacy Data

![Reflect Legacy Data](images/05_reflect_legacy_data)

**그림 3-6: Reflect Legacy Data**

- **시나리오 설명:** 새로운 캐릭터로 플레이하거나 특정 지역 진입 시, 과거 캐릭터의 플레이 기록(Legacy)을 시스템이 동적으로 판독하여 현재 게임 내 환경과 캐릭터 상태에 반영하는 내부 로직이다.
- **참여 객체:** GameManager, WorldState, Region, Event, Character
- **주요 메시지 흐름 및 조건(Alt):**
  1. `GameManager`가 `WorldState.getLegacyData()`를 호출하여 누적된 legacyFlags를 확보한다.
  2. 이를 바탕으로 `Region.updateDescription(worldState)`를 호출하여 지역의 텍스트 설명을 변경한다.
  3. `Event.checkCondition()`을 통해 특수 조건(legacyEffect) 충족 여부를 판단한다.
  4. **[alt: legacyEffect == true]** 조건 만족 시 `Character.applyLegacyResult()`를 호출하여 보너스 능력치나 페널티를 부여한다.
  5. **[alt: legacyEffect == false]** 조건 불만족 시 기존의 기본 상태(keepDefaultState)를 유지한다.

# 4. State Machine Diagram

본 장에서는 시스템 전체를 하나의 추상화된 객체(Game Application)로 간주하고, 사용자의 화면 전환 및 내부 로직 처리에 따른 시스템의 라이프타임 상태 변화를 기술한다.

## 4.1 시스템 전체 상태 전이

| Source State | Trigger (이벤트/조건) | Target State | Action (수행 내용) |
|---|---|---|---|
| [Initial] | 게임 실행 | Main Title | MainMenuUI 렌더링, 리소스 로드 |
| Main Title | New Game 클릭 | Character Select | WorldState 초기화, 시작 프로필 생성 |
| Main Title | Load Game 클릭 | Save/Load | 저장된 슬롯 목록 출력 |
| Save/Load | 저장 파일 선택 | Exploration | SaveData 로드, 이전 위치로 화면 전환 |
| Character Select | 캐릭터 확정 | Legacy Reflection | 선택된 캐릭터에 과거 World 기록 연산 및 동기화 |
| Legacy Reflection | 레거시 연산 완료 | Exploration | 탐험 UI 출력 (텍스트/이미지 갱신) |
| Exploration | 이벤트/오브젝트 상호작용 | Event/Choice | 선택지 UI 출력, 스토리 텍스트 전개 |
| Exploration | 적 조우 | Battle | 전투 화면 전환, BattleManager 초기화 |
| Event/Choice | 특정 NPC/이전 캐릭터 조우 | Party Formation | 생존 조건 체크, 파티 명단에 합류 처리 |
| Event/Choice | 선택 완료 / 보상 획득 | Exploration | WorldState 갱신 후 탐험 복귀 |
| Battle | 적 HP 0 도달 (승리) | Exploration | 명성치 획득, 보상 수령 후 복귀 |
| Battle | 캐릭터 HP 0 도달 (패배) | Game Over/Ending | 캐릭터 사망 기록(Legacy) 저장, 다음 캐릭터 유도 |
| Exploration | 메뉴 - 저장 선택 | Save Complete | 현재 상태 직렬화 후 디스크 쓰기 완료 |
| Save Complete | 확인 버튼 클릭 | Exploration | 이전 탐험 상태로 회귀 |

## 4.2 레거시 시스템이 상태 전이에 미치는 영향

'Echoes of Legacy'의 가장 큰 구조적 특징은 단순히 Exploration ↔ Battle을 반복하는 것이 아니라, 중간에 **Legacy Reflection** 및 **Party Formation** 상태가 개입된다는 점이다.

캐릭터가 사망하거나(Game Over) 특정 스토리를 완료하여 다음 캐릭터로 넘어갈 때, 시스템은 즉각적인 종료 대신 세계의 상태(WorldState)를 업데이트한다. 새 캐릭터를 선택(Character Select)하고 탐험에 진입하기 직전, 시스템은 강제로 **Legacy Reflection State**로 진입하여 이전 회차의 명성치와 선택 결과(Legacy Flags)를 파싱한다. 이 상태 전이를 통해 동일한 맵(Region)이라 하더라도 출현하는 이벤트(Event)나 적의 종류가 변경되는 동적 게임 플레이가 구현된다.

# 5. Implementation Requirements

본 프로젝트는 텍스트와 정적 이미지 중심의 PC 기반 게임으로 기획되었으며, 구현 및 원활한 구동을 위해 다음의 하드웨어 및 소프트웨어 사양을 요구한다.

## 5.1 Hardware Requirements

**[Minimum Specification]**

- **CPU:** Intel Core i3 4세대 이상 또는 AMD 동급 프로세서
- **RAM:** 4GB 이상
- **Storage:** 2GB 이상의 여유 공간 (HDD 또는 SSD)
- **Display:** 1280 x 720 이상의 해상도를 지원하는 모니터

**[Recommended Specification]**

- **CPU:** Intel Core i5 8세대 이상 또는 AMD Ryzen 3 이상
- **RAM:** 8GB 이상
- **Storage:** SSD 내 5GB 이상의 여유 공간
- **Display:** 1920 x 1080 (FHD) 권장

## 5.2 Software Requirements

- **Operating System:** Windows 10 (64-bit) 이상 / macOS 11.0 이상
- **Game Engine:** Unity Engine 2022.3 LTS 이상
- **Implementation Language:** C# (.NET Framework 4.7 이상 호환)
- **IDE (개발 환경):** Visual Studio 2022 또는 JetBrains Rider
- **Version Control:** Git, GitHub Desktop

## 5.3 Nonfunctional Requirements

- **UI 및 시각적 성능 (Usability):** 화려한 3D 그래픽 부하가 없는 텍스트와 정적 이미지 중심의 인터페이스를 사용하여, 권장 사양 이하의 환경에서도 프레임 드랍 없이 원활히 구동되어야 한다. 텍스트 가독성을 최우선으로 배치한다.
- **저장 신뢰성 (Reliability):** 레거시 시스템은 과거 플레이어의 기록 무결성에 전적으로 의존한다. 따라서 SaveData 저장 중 강제 종료가 발생해도 데이터가 오염되지 않도록 임시 파일(Temp) 작성 후 덮어쓰는 안전한 직렬화 로직을 채택해야 한다.
- **모듈성 및 확장성 (Modularity & Scalability):** 캐릭터, 이벤트, 지역 정보는 하드코딩을 배제하고 JSON 형식의 외부 데이터나 Unity ScriptableObject를 활용하여 구조화함으로써, 추후 새로운 캐릭터와 스토리를 코드 수정 없이 쉽게 추가할 수 있어야 한다.
- **응답 시간 (Performance):** 씬(Scene) 전환 및 텍스트 렌더링은 플레이어 입력 후 0.5초 이내에 완료되어 텍스트 어드벤처 특유의 빠르고 직관적인 피드백을 제공해야 한다.

# 6. Glossary

| Term | Definition |
|---|---|
| **Legacy System (레거시 시스템)** | 이전 캐릭터의 선택, 전투 결과, 생존 여부 등의 기록이 다음 캐릭터의 플레이 환경(NPC 대사, 지역 개방 등)에 지속적으로 영향을 미치도록 하는 본 게임의 핵심 시스템. |
| **Reputation System (명성치 시스템)** | 플레이어의 특정 행동이나 전투 승리에 따라 누적되는 점수로, 수치에 따라 특수 이벤트 발동이나 파티 구성 조건으로 작용한다. |
| **Roguelike (로그라이크)** | 절차적 생성, 영구적 죽음 등의 특징을 가지는 롤플레잉 게임의 하위 장르. 본 게임은 이를 차용하되 단일 주인공이 아닌 다중 캐릭터를 통해 죽음을 세계관 내 서사로 연결한다. |
| **WorldState** | 개별 캐릭터의 스탯이 아닌, 게임 세계 전체의 누적된 변화(해방된 지역, 남겨진 아이템, 레거시 플래그 등)를 저장하고 관리하는 전역 도메인 객체. |
| **Unity Engine** | 크로스 플랫폼 게임 엔진. 본 프로젝트에서는 2D UI 및 스크립팅 기능을 활용해 텍스트 기반 인터페이스를 구축하는 데 사용된다. |
| **Turn-based (턴제)** | 전투 시 플레이어와 적이 순서대로 한 번씩 행동(공격, 방어, 스킬 사용 등)을 주고받는 시스템. |
| **Event** | 탐험 중 특정 지역에 도달하거나 오브젝트와 상호작용할 때 발생하는 서사적 사건 단위. |
| **Choice** | 이벤트 발생 시 플레이어에게 주어지는 선택지. 각 선택은 캐릭터 상태 변화와 월드의 레거시 플래그 변경(Consequence)을 동반한다. |
| **Persistence Layer** | 메모리에 존재하는 게임 데이터를 로컬 디스크 파일(SaveData) 구조로 직렬화하여 영구적으로 보존하는 데이터 관리 계층. |
| **Domain Model** | 시스템이 해결하고자 하는 핵심 비즈니스 로직과 데이터(캐릭터, 지역, 적 등)를 캡슐화한 객체 중심의 설계 구조. |
