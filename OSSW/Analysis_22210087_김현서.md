# 1. Introduction

## 1.1 Summary
본 프로젝트는 판타지 세계를 배경으로 하는 턴제 로그라이크 게임 **Echoes of Legacy**의 분석 문서이다. 이 게임은 일반적인 단일 주인공 중심 로그라이크와 달리, 여러 캐릭터를 순차적으로 플레이하면서 이전 플레이의 선택과 결과가 이후 캐릭터의 진행과 세계 상태에 반영되는 **legacy system**을 핵심 특징으로 한다. 플레이어는 각기 다른 배경과 목적을 가진 캐릭터들을 조작하면서 하나의 세계를 여러 시점에서 경험하게 되고, 각 플레이 기록이 누적되어 세계 전체의 흐름을 바꾸게 된다.

## 1.2 Introduce “Echoes of Legacy”
Echoes of Legacy는 텍스트 중심의 탐험, 선택지 기반 이벤트, 턴제 전투, 그리고 다중 캐릭터 진행 구조를 결합한 게임이다. 플레이어는 특정 캐릭터 하나만 계속 육성하는 것이 아니라, 이야기 진행에 따라 새로운 캐릭터를 선택하거나 전환하면서 세계를 탐험한다. 각 캐릭터는 개별적인 능력치와 인벤토리, 진행 위치, 생존 여부를 가지며, 특정 행동이나 전투 결과에 따라 **명성치**를 획득한다. 이 명성치는 특수 이벤트의 발생 조건이 되며, 이전 캐릭터가 남긴 업적이나 선택은 이후 캐릭터의 대사, 지역 상태, 이벤트 분기, 파티 구성 가능 여부 등에 영향을 준다.

본 게임은 고사양 그래픽보다 서사적 연결성과 시스템 구조를 더 중요하게 다루며, 텍스트와 정적 이미지 중심 인터페이스를 통해 학기 프로젝트 범위 내에서 실현 가능한 형태로 설계된다. 이를 통해 플레이어는 반복 플레이 속에서도 단절되지 않는 서사를 경험하고, 개발자는 비교적 현실적인 규모 안에서 독창적인 시스템을 구현할 수 있다.

## 1.3 Goal
이 Analysis document의 목적은 다음과 같다.

첫째, Echoes of Legacy가 플레이어에게 어떤 기능을 제공하는지 use case analysis를 통해 정리한다.  
둘째, 게임 내부에서 관리되어야 하는 핵심 데이터와 개체 관계를 domain analysis를 통해 구조화한다.  
셋째, 실제 플레이 흐름을 고려한 user interface prototype을 제시하여 시스템의 사용 모습을 구체적으로 설명한다.  

이 문서를 통해 최종적으로 본 프로젝트가 어떤 구조로 동작하는 게임인지, 어떤 데이터와 화면 구성이 필요한지, 그리고 핵심 시스템이 서로 어떤 방식으로 연결되는지를 이해할 수 있도록 한다.

# 2. Use case analysis

## 2.1 Use case diagram
본 시스템의 주요 actor는 **Player**와 **Game System**이다. Player는 메뉴 선택, 캐릭터 선택, 탐험, 전투, 저장 등의 직접적 행위를 수행한다. Game System은 이벤트 조건 판정, 전투 결과 계산, 세계 상태 반영, 명성치 누적, 자동 저장 등 내부 로직을 처리한다.

분석 대상 use case는 다음과 같이 정의한다.

- Start New Game
- Load Game
- Select Character
- Explore Area
- Trigger Event
- Make Choice
- Battle Enemy
- Gain Reputation
- Trigger Special Event
- Reflect Legacy Data
- Form Party
- Save Progress

use case diagram을 그릴 때에는 중심에 **Echoes of Legacy System**을 두고, Player actor가 Start New Game, Load Game, Select Character, Explore Area, Make Choice, Battle Enemy, Save Progress와 연결되도록 구성하면 된다. Game System은 Trigger Event, Gain Reputation, Trigger Special Event, Reflect Legacy Data, Form Party, Save Progress와 연관된 내부 처리 주체로 표시하면 된다.

# 3. Domain analysis

## 3.1 Domain overview
Echoes of Legacy의 핵심 도메인은 **캐릭터**, **세계 상태**, **이벤트**, **전투**, **명성치**, **저장 데이터**로 구성된다. 본 게임은 단순히 플레이어 캐릭터 하나의 상태만 관리하는 구조가 아니라, 여러 캐릭터의 진행 기록과 세계 전체의 변화를 함께 다뤄야 한다. 따라서 도메인 모델은 개별 플레이 데이터와 세계 누적 데이터를 동시에 표현할 수 있어야 한다.

## 3.2 Main domain classes

| Domain Class | Description | Main Attributes |
|---|---|---|
| PlayerProfile | 플레이어의 전체 진행 상태를 관리하는 상위 개체 | profileId, currentSaveSlot, playTime |
| Character | 플레이 가능한 인물 정보 | characterId, name, classType, hp, attack, defense, reputation, alive |
| Inventory | 캐릭터가 보유한 자원과 장비 | gold, itemList, equippedWeapon, equippedArtifact |
| Item | 획득 가능한 아이템 정보 | itemId, itemName, itemType, effectValue |
| Region | 탐험 가능한 지역 정보 | regionId, regionName, unlocked, dangerLevel |
| Event | 특정 조건에서 발생하는 사건 | eventId, title, triggerCondition, resultType |
| Choice | 이벤트 안에서 플레이어가 선택하는 분기 | choiceId, description, reward, consequence |
| Battle | 전투 진행 상태 | battleId, enemyId, turnCount, battleResult |
| Enemy | 전투 대상 적 정보 | enemyId, enemyName, hp, attack, pattern |
| WorldState | 세계 전체의 누적 상태 | liberatedRegion, npcStatus, legacyFlags, specialEventFlags |
| ReputationSystem | 명성치 계산 및 이벤트 조건 판정 | totalReputation, reputationRule, threshold |
| SaveData | 저장 파일 단위 데이터 | saveId, timestamp, currentCharacterId, characterStates, worldState |
| Party | 재회한 캐릭터들의 그룹 | partyId, memberList, partyCondition |

## 3.3 Relationship description
1. **PlayerProfile**은 하나 이상의 **SaveData**를 가진다.  
2. **SaveData**는 여러 **Character**의 상태와 하나의 **WorldState**를 포함한다.  
3. 각 **Character**는 하나의 **Inventory**를 가지며, Inventory는 여러 **Item**을 포함한다.  
4. **Character**는 하나 이상의 **Region**을 탐험할 수 있다.  
5. **Region**에서는 여러 **Event**가 발생할 수 있다.  
6. 각 **Event**는 하나 이상의 **Choice**를 포함할 수 있다.  
7. 특정 **Event** 또는 **Region** 진입은 **Battle**을 발생시킬 수 있다.  
8. **Battle**은 하나 이상의 **Enemy**와 연결된다.  
9. **ReputationSystem**은 Character의 행동 결과를 바탕으로 명성치를 누적하고, **WorldState**의 specialEventFlags를 변경한다.  
10. **WorldState**는 legacyFlags를 통해 이전 캐릭터의 업적과 선택을 저장하며, 이후 **Event** 조건 판정에 사용된다.  
11. 특정 조건이 충족되면 여러 **Character**가 **Party**에 편성될 수 있다.  

## 3.4 Domain model interpretation
이 게임에서 가장 중요한 개체는 Character와 WorldState이다. Character는 개별 플레이의 주체이며, WorldState는 여러 플레이 결과가 누적되는 공간이다. 일반 로그라이크는 플레이 종료 시 대부분의 맥락이 초기화되지만, Echoes of Legacy는 WorldState를 통해 이전 캐릭터의 흔적을 유지한다. 따라서 동일한 Region이라도 이전에 누가 어떤 결정을 내렸는지에 따라 Event 구성이 달라질 수 있다.

또한 ReputationSystem은 단순 점수 저장 기능이 아니라 특수 이벤트 해금과 파티 재회 조건을 연결하는 핵심 장치이다. 즉, 전투 결과와 선택지 결과가 곧바로 서사적 변화로 이어지도록 하는 매개 역할을 수행한다.

SaveData는 본 프로젝트에서 매우 중요한 도메인이다. 캐릭터 전환, 이벤트 분기, 사망 여부, 세계 상태 변화까지 모두 보존해야 하기 때문에 저장 구조가 불안정하면 전체 게임 흐름이 무너질 수 있다. 따라서 SaveData는 Character 배열과 WorldState를 함께 포함하는 구조로 설계하는 것이 적절하다.

# 4. User Interface prototype

## 4.1 Main Title Screen
메인 타이틀 화면은 게임 제목, 배경 일러스트, 그리고 주요 메뉴 버튼으로 구성된다. 화면 중앙 또는 우측에는 **New Game / Load Game / Settings / Exit** 버튼이 배치되고, 배경에는 판타지 세계 분위기를 보여주는 대표 이미지가 들어간다. 이 화면의 목적은 게임의 정체성을 전달하고 플레이 시작 경로를 명확히 제공하는 것이다.

**Main components**
- 게임 제목 로고  
- 배경 일러스트  
- New Game 버튼  
- Load Game 버튼  
- Settings 버튼  
- Exit 버튼  

## 4.2 Save / Load Screen
저장 및 불러오기 화면에서는 저장 슬롯 목록, 저장 시각, 현재 플레이 중인 캐릭터 이름, 진행 지역, 누적 명성치를 표시한다. 사용자는 특정 슬롯을 선택해 불러오기 또는 덮어쓰기를 수행할 수 있다. 자동 저장 슬롯은 별도로 구분하여 표시하는 것이 좋다.

**Main components**
- Save slot list  
- Timestamp  
- Current character summary  
- Region / reputation summary  
- Load / Delete / Back 버튼  

## 4.3 Character Select Screen
캐릭터 선택 화면에서는 플레이 가능한 캐릭터의 초상화, 이름, 클래스, 핵심 능력치, 간단한 배경 설명을 보여준다. 잠금된 캐릭터는 실루엣 또는 회색 처리하여 해금 조건을 함께 표시한다. 사용자가 캐릭터를 선택하면 우측 패널에 세부 설명이 나타나고 Start 버튼을 눌러 해당 캐릭터 시점으로 진행한다.

**Main components**
- Character card list  
- Character portrait  
- Class / stat panel  
- Background story preview  
- Start / Back 버튼  

## 4.4 Exploration Screen
탐험 화면은 본 프로젝트의 핵심 UI이다. 상단에는 현재 지역명과 캐릭터 이름, 좌측에는 현재 상황에 대한 텍스트 설명, 우측에는 캐릭터 상태창, 하단에는 행동 선택 버튼을 배치한다. 정적 배경 이미지와 함께 텍스트를 출력하면 비교적 낮은 구현 난도로도 몰입감을 유지할 수 있다.

**Main components**
- Region title  
- Story / description text area  
- Character status panel (HP, reputation, status effect)  
- Action buttons (Move, Investigate, Talk, Rest, Save)  
- Small illustration or background image  

## 4.5 Event / Choice Screen
이벤트 화면은 탐험 화면에서 파생되는 형태로 구성할 수 있다. 중앙 텍스트 영역에 상황 설명과 대사를 출력하고, 하단에 2~4개의 선택지를 제공한다. 특정 선택지는 조건에 따라 비활성화되거나 추가로 표시될 수 있다. 선택 결과는 즉시 다음 텍스트나 보상, 전투, 세계 상태 변화로 이어진다.

**Main components**
- Event title  
- Dialogue / description text  
- Choice buttons  
- Reward or consequence message area  

## 4.6 Battle Screen
전투 화면에서는 아군과 적의 정보를 좌우로 배치하고, 하단에 행동 메뉴를 제공한다. 플레이어는 공격, 방어, 스킬, 아이템 중 하나를 선택하며, 턴 결과는 로그 형태로 텍스트 출력된다. 과도한 애니메이션 없이도 수치 변화와 로그만으로 전투 흐름을 전달할 수 있도록 설계한다.

**Main components**
- Player character info  
- Enemy info  
- HP bars  
- Battle log text area  
- Command buttons (Attack, Defend, Skill, Item)  

## 4.7 Legacy / World Status Screen
본 게임의 차별성을 보여주기 위해, 세계 상태와 이전 캐릭터의 흔적을 요약해서 볼 수 있는 화면이 필요하다. 이 화면에서는 해방된 지역, 사망하거나 생존한 캐릭터, 달성 업적, 누적 명성치, 해금된 특수 이벤트 등을 한눈에 확인할 수 있다. 이 UI는 플레이어가 자신의 선택이 실제로 세계에 축적되고 있다는 사실을 체감하게 만든다.

**Main components**
- World timeline or legacy log  
- Region change summary  
- Character fate list  
- Reputation summary  
- Unlocked event list  

## 4.8 UI design direction
전체 UI는 어두운 판타지 분위기를 유지하되, 복잡한 3D HUD보다는 **텍스트 중심의 명확한 정보 전달**을 우선한다. 따라서 다음 원칙을 따른다.

1. 배경 이미지는 분위기 형성용으로 사용하고, 핵심 정보는 텍스트와 패널 구조로 전달한다.  
2. 버튼 수를 과도하게 늘리지 않고 상황별 핵심 선택만 보여준다.  
3. 캐릭터 정보와 세계 상태를 시각적으로 분리하여 현재 플레이와 누적 결과를 동시에 이해할 수 있게 한다.  
4. 저장, 명성치, 특수 이벤트 해금 여부 등 핵심 시스템 정보는 항상 확인 가능한 위치에 제공한다.

# 5. Glossary

| Term | Description |
|---|---|
| Roguelike | 반복 플레이와 높은 재도전성을 특징으로 하는 게임 장르로, 탐험과 전투, 자원 관리 요소를 포함하는 경우가 많다. |
| Turn-based | 플레이어와 적이 순서대로 행동하는 진행 방식을 의미한다. |
| Legacy System | 이전 플레이의 기록이나 결과가 이후 플레이와 세계 상태에 지속적으로 영향을 주는 구조를 의미한다. |
| Reputation | 플레이어의 행동 결과를 수치화하여 특수 이벤트나 해금 조건에 활용하는 값이다. |
| World State | 지역 해방 여부, NPC 상태, 이벤트 완료 여부 등 세계 전체의 누적 상태를 저장하는 데이터이다. |
| Event Flag | 특정 사건의 발생 여부나 선택 결과를 기록하는 논리 값이다. |
| Party | 두 명 이상의 캐릭터가 함께 행동하는 전투 또는 탐험 단위이다. |
| Save Data | 캐릭터 상태와 세계 상태를 저장하여 이후에 복원할 수 있도록 만든 데이터이다. |
